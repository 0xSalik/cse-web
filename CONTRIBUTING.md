# Contributing

Thank you for your interest in contributing.

## Getting Started
1. Fork the repository
2. Create a new branch (`feature/your-feature-name` or `fix/issue-name`)
3. Make your changes
4. Commit with clear and concise messages
5. Push your branch to your fork
6. Open a Pull Request

## Pull Request Guidelines
- Ensure your code follows existing structure and conventions
- Keep PRs focused and minimal
- Provide a clear description of changes
- Include screenshots for UI-related changes if applicable

## Review Process
All pull requests are reviewed by the project lead.

Final approval and merge authority rests with the lead developer, Salik Khan.

## Notes
- Large or breaking changes should be discussed via an issue before implementation
- Incomplete or low-quality PRs may be rejected
